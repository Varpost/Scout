Your AI security team in a CLI. Find vulnerabilities before hackers do — free, local, no signup.

The Problem

AI coding assistants write insecure code. Hardcoded secrets, SQL injection, missing auth — they ship it all. You can't afford a security team. But you can't afford to get hacked either.

Scout catches what AI introduces. Static analysis finds the most common mistakes AI assistants make — leaked keys, string-built SQL, shell=True, missing security headers — no API keys needed. Then it hands your own AI assistant ready-to-paste fix prompts.

Quick Start

Install

# zero-install with uv:
uvx scout-security scan .

# or install permanently:
pip install scout-security

Scan your project

scout scan ./my-project

Read the report

# Opens security-report.md with:
# - Every vulnerability explained in plain English
# - Severity ratings (CRITICAL → LOW)
# - Exact fix instructions
# - Phased remediation plan (fix safe things first)

Three Ways to Use the Results

One scan, three output formats — pick with --format. Scout finds the problem; your own AI fixes it.

1. Markdown report (default)

A clean, human-readable report you open and read.

scout scan ./my-app
scout scan ./my-app -o security-report.md

2. AI-ready prompts

Ready-to-paste prompts you drop into Cursor, Claude, or Copilot — your AI already knows your codebase, so it writes the best fix.

scout scan ./my-app --format ai-prompt          # writes security-prompts.md
scout scan ./my-app --format ai-prompt -o prompts.md

3. Machine-readable JSON

Structured output for piping into agentic tooling, CI, or your own scripts.

scout scan ./my-app --format json               # prints JSON to stdout
scout scan ./my-app --format json -o report.json
scout scan ./my-app --format json | jq '.findings[]'

Using Scout with Your AI Assistant

The loop: Scout finds deterministically → your AI fixes → Scout re-verifies. Same scan, same findings, zero tokens every pass — re-checking a fix never costs inference. Pick the surface that matches how you work.

1. You + a chat assistant (Claude, Cursor, Copilot Chat)

Generate self-contained fix prompts, paste one into your assistant, then rescan to confirm the finding is gone.

scout scan . --format ai-prompt   # writes security-prompts.md
# paste a prompt into your assistant, let it fix, then:
scout scan .                      # clean? the loop is closed

2. An agent that calls Scout itself (Claude Code, Cursor Agent)

Run Scout as an MCP server your agent invokes in a scan → fix → rescan loop — the deterministic, zero-token verifier inside the loop.

Claude Code — install the plugin (bundles the server; requires uv):

/plugin marketplace add Varpost/Scout
/plugin install scout@scout

Any other MCP host — Cursor (.cursor/mcp.json), Claude Desktop (claude_desktop_config.json), Cline (cline_mcp_settings.json), Windsurf (~/.codeium/windsurf/mcp_config.json), VS Code (.vscode/mcp.json, under a "servers" key) — all take the same definition:

{
  "mcpServers": {
    "scout": {
      "command": "uvx",
      "args": ["--from", "scout-security[mcp]", "scout-mcp"]
    }
  }
}

No uv? pip install "scout-security[mcp]" and use "command": "scout-mcp" with no args.

Then tell the agent: "Scan this project with Scout, fix every finding, then scan again — repeat until it reports zero." It calls scan_path, applies fixes, and calls again.

3. CI / pre-commit (make the loop mandatory)

Turn the loop into a gate — the build fails until findings are fixed.

scout scan . --fail-on high       # exit 1 on HIGH+ findings

Features

🔍

Static Analysis

Regex-based patterns detect secrets, injections, XSS, missing headers, and vulnerable deps. Deterministic: same scan, same findings.

🤖

AI-Ready Output

--format ai-prompt turns every finding into a self-contained fix prompt for Cursor, Claude, or Copilot — your AI already knows your codebase.

📋

Phased Fix Plan

Phase 1: zero-risk (secrets, deps). Phase 5: architecture. Fix safely, ship confidently.

🔒

Privacy First

Your code never leaves your machine. No telemetry, no signup. Dependency checks query the OSV.dev database with package names only — never your code.

⚡

Fast

Scans 10,000 lines in under 10 seconds. No build step, no config file needed.

🧩

Extensible

Write a new scanner in 30 lines of Python. Inherit BaseScanner, implement scan_file(), done.

Built-in Scanners

ScannerDetectsSeverity
secrets AWS keys, GitHub tokens, Stripe keys, DB URLs, private keys, JWT secrets, passwords CRITICAL
injection SQL injection (concat, f-strings, format), command injection, eval() CRITICAL
headers Missing security headers (Express/Flask/Django/FastAPI), wildcard CORS, missing CSRF LOW–MEDIUM
deps Known vulnerabilities in pinned pip dependencies (via OSV.dev) HIGH

CLI Commands

scout scan

scout scan <path> [OPTIONS]

Options:
  --format, -f   Output format: markdown | ai-prompt | json | sarif (default: markdown)
  --output, -o   Output path (omit with --format json to print to stdout)

Optional: AI Confirmation Pass

The core scan is always static, deterministic, offline, and zero-token — same scan, same findings, no API key. That is the default and never changes.

Optionally, Scout can send only the flagged snippet (never whole files) of each heuristic finding to an AI provider (Anthropic, OpenAI, or local Ollama), which can downgrade or dismiss a false positive. Dependency and project-level findings are deterministic facts and are never second-guessed; any provider error leaves findings untouched (fails open), so it can never hide a real issue. It is off by default — enable per run with --model anthropic, --model openai, or --model ollama.

Add Your Own Scanner

from scout.scanners import register_scanner
from scout.scanners.base import BaseScanner
from scout.models import Finding
from pathlib import Path

@register_scanner
class MyScanner(BaseScanner):
    name = "my-scanner"
    description = "Detects my custom pattern"

    def scan_file(self, file_path: Path, content: str) -> list[Finding]:
        findings = []
        # Your detection logic here
        return findings

Then add one import line in scout/scanners/__init__.py. That's it.
